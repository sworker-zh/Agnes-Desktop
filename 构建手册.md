## 二、构建 Windows 安装包

### 2.1 一条命令构建

```bash
npm run tauri build
```

- **首次构建**需要 3-5 分钟（下载编译 Rust 依赖）
- **后续增量构建**只需 10-30 秒

### 2.2 输出文件

```
src-tauri/target/release/bundle/
├── nsis/
│   └── Agnes AI Tool_0.1.0_x64-setup.exe     ← NSIS 安装器（推荐分发）
├── msi/
│   └── Agnes AI Tool_0.1.0_x64_en-US.msi      ← MSI 安装包（企业部署）
src-tauri/target/release/
└── agnes-ai-tool.exe                           ← 免安装版（直接运行）
```

| 格式 | 说明 | 适用场景 |
|---|---|---|
| `.exe` (NSIS) | 标准安装器，体积更小，支持自动更新 | 日常分发 |
| `.msi` | Windows Installer 格式 | 企业批量部署 |
| `agnes-ai-tool.exe` | 独立可执行文件，免安装 | 便携使用 |

## 三、常用命令速查

| 操作 | 命令 |
|---|---|
| 完整构建（.exe + .msi） | `npm run tauri build` |
| 只构建不打包（快速测试） | `npm run tauri build -- --no-bundle` |
| 只生成 .exe | `npm run tauri build -- --bundles nsis` |
| 只生成 .msi | `npm run tauri build -- --bundles msi` |
| 桌面端开发（热更新） | `npm run tauri dev` |
| 只构建 Web 前端 | `npm run build` |
| Web 开发模式 | `npm run web:dev` |
| Web 生产模式（本地托管） | `npm run web` |

## 四、自定义配置

编辑 `src-tauri/tauri.conf.json`：

```json
{
  "productName": "Agnes AI Tool",
  "version": "0.1.0",
  "identifier": "com.agnes-ai.tool",
  "bundle": {
    "targets": "all",
    "icon": [
      "icons/32x32.png",
      "icons/128x128.png",
      "icons/128x128@2x.png",
      "icons/icon.ico"
    ]
  }
}
```

| 字段 | 说明 |
|---|---|
| `productName` | 安装包和应用窗口显示的名称 |
| `version` | 版本号，每次发版修改 |
| `identifier` | 应用唯一标识（反向域名格式） |
| `bundle.targets` | `"all"` = NSIS + MSI；`["nsis"]` 只生成 .exe |

## 五、更换应用图标

替换 `src-tauri/icons/` 目录下的文件：

```
icons/
├── icon.ico           ← Windows 图标（256x256）
├── icon.png           ← 主图标（1024x1024，源文件）
├── 32x32.png
├── 128x128.png
├── 128x128@2x.png
├── icon.icns          ← macOS 图标
└── ...（其他尺寸自动裁切）
```

**推荐工具：** 准备一张 **1024×1024** 的 PNG 图片，用以下命令自动生成全套图标：

```bash
npm run tauri icon path/to/your-icon.png
```

## 七、常见问题

### Q: 构建报错 `failed to run custom build command`

**原因：** VS Build Tools 未安装或未勾选 C++ 工作负载。

**解决：** 安装 Visual Studio Build Tools 2022，勾选 "Desktop development with C++"。

### Q: 构建报错 `Permission xxx not found`

**原因：** `capabilities/default.json` 中使用了不存在的权限名。

**解决：** 检查 `src-tauri/capabilities/default.json`，只使用 Tauri 2.0 支持的权限。

### Q: 首次构建很慢

**原因：** Rust 编译需要下载和编译所有依赖（Tauri 框架、插件等）。

**解决：** 正常现象，首次 3-5 分钟。后续增量构建只编译改动部分，速度很快。

### Q: 想只构建前端测试 UI

```bash
npm run build          # 输出到 dist/
npm run web:dev        # 开发模式，浏览器访问 localhost:3000
```

### Q: 免安装版 .exe 在哪里

```
src-tauri/target/release/agnes-ai-tool.exe
```

可以直接复制到任何 Windows 电脑运行，无需安装。
